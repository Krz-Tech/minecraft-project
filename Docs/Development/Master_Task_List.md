# 全体タスク・ロードマップ (Project Road Map & Task List)

本プロジェクト「Code::Playground」の完遂に向けた、フェーズ別の実装タスクを整理します。

---

## Phase 1: コア戦闘システムの完成 (Core Combat Completion)
物理演算やEN管理などの基礎部分は実装済みですが、視覚的なフィードバックとバランス調整が必要です。

- [ ] **FCS View Trackingの実装**:
    - [ ] `fcs-view.sk` の有効化と Paper API を用いたスムーズな視点追従。
    - [ ] 敵の移動速度に応じた補正精度の動的な変化。
- [ ] **リッチな戦闘HUDの実装**:
    - [ ] リソースパックのカスタムフォントを用いた **EN/AP ゲージ** の表示（経験値バー/アクションバー）。
    - [ ] ロックオン中の敵に対する **空間投影ターゲットマーカー**（Text Display パケット送信）。
- [ ] **ダメージインジケーターの高度化**:
    - [ ] ダメージの **累積表示（コンボ）** ロジックの実装（`480 (+120)` 形式）。
    - [ ] クリティカル、ダイレクトヒットの視覚的な差別化。

## Phase 2: アイテム・装備データベースの統合 (Item & Equipment Integration)
`Docs/ItemDatabase/` の定義とスクリプト変数を完全に紐付けます。

- [ ] **Tech Suit ステータス反映**:
    - [ ] 装備中のスーツに応じた `max_en`, `stagger_threshold`, `speed` 等の自動計算。
- [ ] **武器属性の実装**:
    - [ ] **EN武器**: 射撃時に一定の EN を消費するロジック。
    - [ ] **物理武器**: 高い衝撃（Impact）を与え、スタッガーを誘発。
- [ ] **FCSチップの換装**:
    - [ ] 装備するチップによって `lock_speed`, `max_distance`, `reticle_angle` が変化する仕組み。

## Phase 3: プレイグラウンドの拡張 (Playground Expansion)
サバイバル・探索・帰還のゲームループを完成させます。

- [ ] **エネミー実装 (MythicMobs連携)**:
    - [ ] α, β, γ クラスの敵定義と Skript 側でのドロップ・報酬判定。
    - [ ] 特定エリア（危険地帯）での湧き調整。
- [ ] **環境ギミック**:
    - [ ] 天候（雨・雷）によるスタミナ（EN）消費増大や、一部Tech装備の機能不全。
- [ ] **リワード・換金システム**:
    - [ ] 帰還（`/vanish`）成功時に **Non-Scopeアイテム** を自動で Δコインに変換する処理。

## Phase 4: 経済と拠点システム (Economy & Housing)
RPGとしての永続的な進行要素を構築します。

- [ ] **統合メニュー GUI の完成**:
    - [ ] プロフィール、装備管理、経済、拠点の各サブメニューの実装。
- [ ] **拠点の土地管理 (Housing)**:
    - [ ] 区画の購入、テレポート、権限管理ロジック。
    - [ ] **デリバリーポスト** 実装（帰還アイテムの受け取り。

## 優先順位 (Critical Tasks)
1. **FCS View Tracking & Rich HUD**: 3人称視点でのプレイ感の基盤となるため最優先。
2. **Item-Stat Synchronization**: 装備によるキャラクターの差別化を早期に確立。
3. **Enemy AI & Loot**: プレイグラウンドでの「遊び」のメインコンテンツ。
