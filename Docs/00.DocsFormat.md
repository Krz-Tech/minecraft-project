# ドキュメントフォーマット規約

本ドキュメントでは、プロジェクト全体で統一されたMarkdownドキュメントの記述形式を定義します。全てのドキュメント作成時はこの規約に従ってください。

## 1. ファイルヘッダー

全てのドキュメントは以下のヘッダーで開始してください。

```markdown
ファイル：`Docs/カテゴリ/ファイル名.md`
更新日時：`YYYY/MM/DD`
```

* ファイルパスはリポジトリルートからの相対パスを記述
* 更新日時は最終更新日を `YYYY/MM/DD` 形式で記述

## 2. タイトルと概要

* ヘッダー直後に `# タイトル` (H1) を記述
* タイトル直後にドキュメントの概要を1〜2文で記述
* 概要は本文との区切りとして機能するため必須

## 3. セクション構成

### 見出しレベル

| レベル | 用途 | 例 |
|:---|:---|:---|
| **H1** `#` | ドキュメントタイトル（1つのみ） | `# ドキュメントフォーマット規約` |
| **H2** `##` | 主要セクション（番号付き） | `## 1. ファイルヘッダー` |
| **H3** `###` | サブセクション | `### 見出しレベル` |
| **H4** `####` | 詳細項目 | `#### 補足事項` |

### セクション番号

* H2レベルのセクションには `## 1. セクション名` の形式で番号を付与
* 番号は1から連番で割り当て
* H3以降のサブセクションには番号不要

## 4. リスト記法

### 箇条書き

* `*` を使用（`-` は使用しない）
* ネストする場合は4スペースでインデント

```markdown
* 項目1
* 項目2
    * ネスト項目2-1
    * ネスト項目2-2
* 項目3
```

### 番号付きリスト

* 手順や順序が重要な場合のみ使用
* `1.`, `2.` の形式で記述

## 5. テーブル記法

* ヘッダー行の項目は **太字** で記述
* 左揃え `:---` を基本とする

```markdown
| カテゴリ | 技術 | 用途 |
|:---|:---|:---|
| **Compute** | Kubernetes | コンテナオーケストレーション |
| **Storage** | PVC | 永続ストレージ |
```

## 6. コードブロック

* 言語指定付きのフェンスドコードブロックを使用
* インラインコードは `` ` `` で囲む

```markdown
```skript
on join:
    send "Welcome!" to player
`` `
```

* 対応言語例: `markdown`, `skript`, `yaml`, `json`, `mermaid`, `text`

## 7. ダイアグラム

* Mermaid記法を使用してフローチャート・シーケンス図を記述

```markdown
```mermaid
graph LR
    A[Start] --> B[Process]
    B --> C[End]
`` `
```

## 8. リンクと参照

### 内部リンク

* 相対パスでドキュメントを参照

```markdown
詳細は [開発フロー](Docs/0.System/0.Sys_Flow.md) を参照してください。
```

### セクション内リンク

* アンカーリンクを使用

```markdown
[参照ドキュメント](#1-参照ドキュメント) を確認してください。
```

### 外部リンク

* 完全なURLを使用

```markdown
* [Skript Documentation](https://docs.skriptlang.org/)
```

## 9. 強調と注意

### テキスト強調

| 記法 | 用途 | 例 |
|:---|:---|:---|
| `**太字**` | 重要な用語・キーワード | **Kubernetes** |
| `` `コード` `` | コマンド・変数・パス | `./mc start` |

### 注意書き

* 引用記法 `>` を使用

```markdown
> **注意**: この操作は元に戻せません。
```

## 10. 禁止事項

* H1の複数使用
* 番号なしのH2セクション（セクション番号は必須）
* `-` による箇条書き（`*` を使用すること）
* 言語指定なしのコードブロック
* 絶対パスでの内部リンク
