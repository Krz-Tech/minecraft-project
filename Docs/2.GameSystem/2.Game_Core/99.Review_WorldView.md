# 世界観レビュー

本ドキュメントでは、現時点で定義された世界観設定について、一貫性・独自性・拡張性・課題の観点からレビューを行います。

## 1. 世界観の全体像

### 構成要素

| カテゴリ | 要素 | 状態 |
|:---|:---|:---|
| **歴史** | The Peaceful Update、2人の伝説的プレイヤー | 定義済 |
| **ディメンション** | オーバーワールド、ネザー、エンド、プレイグラウンド | 定義済 |
| **社会構造** | 各ディメンションの役割、統治機関 | 定義済 |
| **組織** | Delta Guild | 定義済 |
| **経済** | Δコイン、ディメンション間貿易 | 概要のみ |
| **技術** | エンダーコンパス、ポータル技術 | 定義済 |

### 核となるコンセプト

1. **The Peaceful Update**: 優しさで世界を救った歴史
2. **プレイグラウンド**: 忘れ去られたスナップショット（メタ設定）
3. **Delta Guild**: 伝説のプレイヤーの遺産を継ぐ組織
4. **2人の伝説的プレイヤー**: 世界最大の謎

## 2. 評価

### 強み

#### Minecraftらしさの維持

* 既存のモブ・ディメンション・アイテムを活用
* 「Peaceful」「スナップショット」などMinecraft用語のメタ的活用
* バニラMinecraftを遊んだプレイヤーに馴染みやすい

#### 独自性のある設定

* 「全モブが友好化した平和な世界」という逆転の発想
* プレイグラウンド＝バグスナップショットというメタ設定
* 敵対モブの能力は「技術力」という合理的な解釈

#### ストーリーの余白

* 2人の伝説的プレイヤーの謎
* プレイグラウンド出現と失踪の関連性
* ネザー・エンド辺境の不穏な集団
* これらはイベント・アップデートで展開可能

#### Extraction RPGとの整合性

* プレイグラウンドが「唯一の危険地帯」である必然性
* Delta Guildによる組織的サポートの説得力
* エンダーコンパスによる救助システムの合理性

### 懸念点・検討事項

#### 設定の複雑さ

* 新規プレイヤーが理解するには情報量が多い
* チュートリアルでどこまで説明するかの設計が必要
* ドキュメントは詳細だが、ゲーム内での伝え方は別途検討

#### 未定義の要素

| 要素 | 状態 | 優先度 |
|:---|:---|:---|
| 経済システムの詳細（Δコインの相場感） | 未定義 | 高 |
| ランクシステムの数値 | 未定義 | 高 |
| 帰還条件の具体値（時間・距離） | 未定義 | 高 |
| プレイグラウンドのマップ生成 | 未定義 | 中 |
| PvPルールの詳細 | 未定義 | 中 |
| 限定アイテムの種類 | 未定義 | 中 |
| オーバーワールド中心地の詳細 | 未定義 | 低 |
| NPCキャラクターの設定 | 未定義 | 低 |

#### 潜在的な矛盾・要確認点

1. **古代都市のポータル vs ギルド本部** ✅ 解決済
   * ギルド本部は古代都市の「直上」に建設された
   * 古代都市の研究開発を目的として建築
   * ギルド本部を中心に自然発生的に中心都市が形成

2. **ウィザーの設定** ✅ 解決済
   * ウィザーはネザーに元から存在する設定
   * ネザーの創世から存在する古代の統治者として明記

3. **クラフターの立場** ✅ 解決済
   * モブたちはプレイヤーを「プレイヤー」として本能的に認識
   * Minecraftの世界の一部として、遺伝子的に組み込まれている
   * 「世界のパーツの1つ」として自然に受け入れられる存在

## 3. 一貫性チェック

### 整合が取れている設定

* The Peaceful Update → 全モブ友好化 → プレイグラウンドのみ敵対
* 2人目のプレイヤーの研究 → Delta Guild → ポータル技術 → プレイグラウンド探索
* エンドの技術力 → エンダーコンパス開発 → 救助システム
* ウィザーの三権分立 → ネザーの秩序ある社会 → エネルギー供給のインフラ役割

### 要調整の設定

| 項目 | 現状 | 状態 |
|:---|:---|:---|
| ポータルの所在地 | ギルド本部（地上）→ 古代都市（地下）→ ポータル | ✅ 解決済 |
| 古代都市の役割 | ポータル設置場所 + ギルド本部建設の理由 | ✅ 解決済 |

## 4. 拡張性評価

### 高い拡張性を持つ要素

| 要素 | 拡張可能性 |
|:---|:---|
| **2人の伝説的プレイヤーの謎** | メインストーリー、イベント、新エリア解放などに活用可能 |
| **辺境の不穏な集団** | レイドイベント、新勢力、ストーリー展開に活用可能 |
| **プレイグラウンド** | 新エリア、新ボス、シーズンイベント等を追加可能 |
| **Delta Guild** | 新部門、新NPC、サブクエスト等を追加可能 |
| **ディメンション間貿易** | 経済イベント、新アイテム導入の口実として活用可能 |

### 将来的なコンテンツ候補

1. **ストーリーイベント**: 伝説のプレイヤーの手がかり発見
2. **レイドイベント**: ネザー/エンド辺境の反乱勢力との戦闘
3. **新ディメンション**: プレイグラウンドの深層、または新たなバグ空間
4. **ギルドランク報酬**: ネザライトランク到達で伝説に関する情報解放
5. **季節イベント**: 各ディメンションの祭り・行事

## 5. 提案

### 短期（優先度高）

1. **ポータル所在地の統一** ✅ 解決済
   * ギルド本部（地上）→ 古代都市（地下）→ ポータルの構造で統一
   * ギルド本部は古代都市の研究開発のために直上に建設
   * 中心都市はギルド本部を中心に自然発生的に形成

2. **チュートリアルでの世界観導入設計**
   * 何を伝え、何を隠すかの方針決定
   * ゲーム内での情報開示方法（NPC会話、本、掲示板など）

3. **経済システムの数値設計**
   * Δコインの基準価値
   * アイテム売却相場の目安
   * ランクアップに必要な探索回数の目安

### 中期（優先度中）

1. **プレイグラウンドの詳細設計**
   * マップ生成ルール
   * 出現モブの種類・強さ
   * 限定アイテムの設計

2. **NPCキャラクターの設定**
   * ギルド受付のヴィンディケーター
   * 各部門の代表モブ

### 長期（優先度低）

1. **ストーリーラインの設計**
   * 伝説のプレイヤーの真相をどこまで明かすか
   * イベントでの情報開示スケジュール

## 6. 総評

### 評価: A（優れた基盤）

現時点の世界観設定は、以下の点で高く評価できます：

* **Minecraftらしさと独自性のバランス**が良い
* **Extraction RPGとしての必然性**が世界観で説明されている
* **ミステリー要素**が今後のコンテンツ展開の余地を残している
* **各設定間の整合性**が概ね取れている

一方で、以下の点は今後の課題です：

* 一部設定の**統一・明確化**が必要（ポータル所在地など）
* **数値設計**（経済、ランク、帰還条件）が未定義
* ゲーム内での**情報伝達方法**の設計が必要

全体として、世界観の骨格は十分に固まっており、ゲームシステムの詳細設計に進む準備が整っています。
