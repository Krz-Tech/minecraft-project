# 戦闘HUD・アクションバー表示仕様 (Combat HUD & Action Bar Specifications)

## 概要
戦闘中の「直感的なリソース管理」と「空間把握」を支援するUI仕様です。
テックスーツのバイザーが情報を解析し、網膜に直接投影しているという設定に基づきます。

---

## 1. アクションバー (Action Bar: Resource Monitor)
プレイヤーの手元（ホットバーの上部）に常時表示される、生命線となるリソース表示です。

### 表示形式
```
&fAP [&a||||||||||&f] &8| &fEN [&b||||||||||&f] &8| &fFCS: &7READY
```

### 詳細ルール
| 項目 | 色の遷移 | 変化ロジック |
| :--- | :--- | :--- |
| **AP (Suit Integrity)** | `&a` (100-50%) -> `&e` (50-25%) -> `&c` (<25%) | スーツの耐久値に連動。0で非表示または警告。 |
| **EN (Energy)** | `&b` (通常) -> `&7` (スタッガー停止中) | 使用中やチャージ中にリアルタイムに伸縮。 |
| **FCS Status** | `&7` (捜索中) -> `&b` (捕捉) -> `&c` (ロックオン) | ターゲットとの距離やロック状態を表示。 |

---

## 2. ダメージインジケーター (Spatial HUD: Damage)
攻撃命中時、対象の傍にポップアップする数値表示です。

### 累積表示ロジック (Cumulative Display)
短時間の連射攻撃において視認性を高めるため、以下の挙動を採用します。

1.  **初撃**: 対象の座標に `TEXT_DISPLAY` をパケット生成。 (例: `120`)
2.  **追撃**: 2秒以内に同じ対象へ攻撃した場合、既存の表示を更新。
    *   表示: `累積ダメージ (+今回のダメージ)`
    *   例: `480 (+120)`
3.  **消失**: 最後の攻撃から2秒経過、または対象の死亡でフェードアウト。

---

## 3. スタッガーゲージ (Spatial HUD: Stagger)
ターゲットの状態を可視化する、ロックオン対象の頭上に表示されるゲージ。

### ビジュアル
```
&7[ &f||||&8|||||| &7]
```
*   **蓄積中**: 白色のバーが右に伸びる。
*   **フル蓄積**: 全体が赤く点滅し、ゲージ直上に `&c&l!! STAGGER !!` を表示。
*   **スタッガー中**: ゲージが赤く染まり、復帰までの時間をカウントダウン。

---

## 4. ナビゲーション・ロックオン (Spatial HUD: Targeting)
FCS（火器管制）による索敵情報の投影。

*   **ターゲットマーカー**: ロックオンした敵を `&b[ ]` で囲む。
*   **Vanish Point**: 帰還可能地点を空間内に `&b▲ VANISH POINT` として表示。
*   **距離計**: マーカー下部に `32m` のように距離を併記。

---

## 5. 演出・ウェイト
*   **パケット制御**: サーバー負荷軽減のため、表示範囲外（40-64ブロック以上）のインジケーターはパケットを送信しない。
*   **フォント**: バイザー感を出すため、可能であればリソースパックによる等幅フォント（Cyberpunk Style）の使用を想定。
