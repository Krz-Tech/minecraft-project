## ✅ 評価：非常に明確でユニークなシステムです！

Tarkovライクな「出撃・収集・帰還」のループがMinecraftの要素とうまく融合しており、非常に面白そうです。特に**Scope / Non-Scope**による換金システムは、経済バランスを保つ上で非常に効果的だと感じました。

### 🌟 素晴らしい点 (Pros)

1.  **Scope / Non-Scope の概念**
    *   バニラアイテム（土、石など）を強制的に換金（Non-Scope）し、特定素材（Scope）のみ持ち帰れる仕様は、メインサーバーの経済・インベントリ崩壊を防ぐ**素晴らしい発明**です。
    *   これにより「建築にはコインが必要」という経済循環が生まれます。

2.  **エンティティのクラス分け (α, β, γ)**
    *   **α（中立）**: 初心者でも狩りやすい、リスク管理の選択肢。
    *   **β（敵対）**: 中級者の目標。
    *   **γ（強敵・感知範囲外）**: 上級者向け、またはステルス要素としての緊張感。
    *   役割分担が明確で、プレイヤーの実力に合わせた立ち回りが可能です。

3.  **ブロック破壊からのScopeドロップ**
    *   戦闘だけでなく「採掘」にも価値を持たせている点がMinecraftらしくて良いです。

4.  **死亡ペナルティのバランス**
    *   「持ち込みアイテム維持」「ランク経験値減少」というペナルティは、**「挑戦はしやすいが、死ぬと悔しい」**という絶妙なバランスです。全ロス（Tarkov方式）はMinecraftだとストレスが高すぎるため、この調整は正解だと思います。

## 🔧 改善・検討の提案 (Suggestions)

実装やゲームバランスの観点から、いくつか検討事項を挙げさせていただきます。

### 1. マッチング・占有問題への対策
**現状の懸念:** 「全プレイヤー共通のワールド」とあるため、γクラスのようなレア敵や、美味しい資源エリアの取り合い（Spawn Camping）が発生する可能性があります。
*   **提案:**
    *   スポーン位置を分散させるアルゴリズムの工夫（敵プレイヤーから一定距離離すなど）。
    *   資源リセットの仕組み（一定時間でチェストや鉱石が復活するなど）。

### 2. "Scope" アイテムの視認性
**現状の懸念:** インベントリ内でどれがScope（持ち帰り可）でどれがNon-Scope（換金）か分かりにくいと、捨てる/拾うの判断にストレスがかかります。
*   **提案:**
    *   Scopeアイテムには特定のLore（説明文）や、名前の色（例: <span style="color:cyan">Scope Item</span>）を付けて視認性を高める。

### 3. 「3人称視点強制」の実装
**現状の懸念:** Minecraft本体の設定で視点を変更できるため、サーバー側で強制・制御するのは技術的に少し工夫が必要、またはMod/クライアント側の協力が必要になる場合があります。
*   **提案:**
    *   プラグインで視点変更パケットを監視・強制戻し（酔う可能性あり）。
    *   あるいは「1人称だとデバフ（盲目など）がかかる」などのソフトな制約にする。
    *   `一部アクセサリ` で1人称可能という仕様は面白いので、ぜひ実装したいところです。

### 4. 帰還条件のUI表示
**現状の懸念:** 「あと何ブロック移動？」「あと何分？」が分からないと不安になります。
*   **提案:**
    *   ActionBar（ホットバーの上）やBossBarで「帰還可能まで: 残り500m / 5分」のように常に表示する。
